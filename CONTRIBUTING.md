# CONTRIBUTING

## Ветки
- `feature/<scope>` — фичи
- `bugfix/<scope>` — баги
- `hotfix/<scope>` — срочные правки
- `chore/<scope>` — рутинные изменения
- `release/<version>` — релизы

## Коммиты
- Кратко: `feat: add vote to point`, `fix: nil pointer in login`.
- Связывайте с задачами: `Closes #123`.

## Процедура PR (code review)
1. Откройте PR в `main` с описанием, чеклистом, ссылкой на задачу.
2. Назначьте ≥1 ревьюера.
3. Исправьте замечания, запросите повторное ревью.
4. Мёрдж через **Squash & merge**.

## Стандарты кода
### Go
- Обязателен `gofmt`.
- Экспортируемые элементы комментируются.
- Ошибки возвращаем и логируем (`log.Printf` / logger).

### JS/HTML
- Логика в отдельных модулях в `web/static/js`.
- Названия переменных — по смыслу; минимум «магических чисел».
