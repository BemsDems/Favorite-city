# CONTRIBUTING

## Ветки
- `feature/<scope>` — фичи
- `bugfix/<scope>` — баги
- `release/<version>` — релизы

## Коммиты
- Кратко: `feat: add vote to point`, `fix: nil pointer in login`.
- Связывайте с задачами: `Closes #123`.

## Процедура PR (code review)
1. Откройте PR в `main` с описанием, чеклистом, ссылкой на задачу.
2. Назначьте ≥1 ревьюера.
3. Исправьте замечания, запросите повторное ревью.
4. Мёрдж через **Squash & merge**.

##Стандарты кода и комментариев
- Go (backend):
•	Именование по стандарту Go (CamelCase для экспортируемых, err для ошибок).
•	Комментарии на функции публичного API в стиле // Name ....
•	Структура пакетов: слои api, auth, storage.
•	Ошибки оборачиваем и логируем; никаких «паник» в обработчиках.

- JavaScript/HTML (frontend):
•	Простая модульность: отдельные файлы в web/static/js.
•	Комментарии только там, где не очевидно; именование переменных по смыслу.
•	Никаких «магических чисел» — выносим в константы.
